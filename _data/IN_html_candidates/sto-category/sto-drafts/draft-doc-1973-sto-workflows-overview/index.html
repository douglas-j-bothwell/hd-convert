Orchestrated Scans

  • Fully configurable in one one Security step
    • Runs the scan and ingests the detected issues
  • Easiest workflow -- good for running scans with default settings
  • Configure the Security step with the following info:
    • The scanner and scan settings
    • The object to scan. The info you provide depends on the object type:
      • Container image => image type, domain, owner, project, tag
      • Code repo => project and branch
      • Instance (website) => identifier, environment, domain, path, protocol, port
    • Access credentials to the domain hosting the object to scan.
  • Example:
    The Set Up Harness for STO topic describes how to scan a codebase repo using Bandit. The banditScan step specifies all the information needed to scan the object:
    • The scanner and scan settings
    • Th object to scan: object type (repository) and the repo name and branch. The Git account URL is specified in the Codebase object.

Ingestion-Only Scans

  • The Security Step ingests issues generated elsewhere
  • Requires an additional Run step to provide the issues data to ingest
  • The step runs a scan or pulls issues data generated outside the pipeline
  • Good for supported scanners with custom settings
  • Also good for ingesting issues from unsupported scanners
  • Example:
    In the following pipeline, the scan process consists two steps:
    • The first step runs Trivy scan on an image and outputs the results to a local JSON.
      To implement this, you create a Run step and then specify the bash commands to install Trivy and run the scan. The actual command would look something like this:
      trivy -q image --timeout 20m -f json -o /shared/customer_artifacts/aqua.json <image>:<tag>
    • The second step ingests the scan results.
      To implement this, you create a Security step and configure it with the following information:
      • The scanner and scan settings
      • The path and filename of the issues data file
      • Information about the scan and the scanned object: the object type (local image) and the container project and tag

Data-Load Scans

  • A subcategory of orchestrated scans
  • Instead of running a scan, the Security step pulls issues from an external source
  • Good for tools that support downloading data via an API endpoint
