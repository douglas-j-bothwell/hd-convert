Cloud cost anomaly detection can be used as a tool to keep cloud costs under control. You can specify a communication strategy to receive alerts for each anomaly that is detected. The anomaly alert can be sent to a Slack channel, Email, and Microsoft Teams.

Before You Begin

  • Detect Cloud Cost Anomalies with CCM

Set Up Slack Notifications

You can receive your alert report for the cost anomalies in your Slack channel. You simply need to add a Slack Incoming Webhook URL in Harness. Perform the following steps to configure your Slack channel: 

  1. In Cloud Costs, click Anomalies.
  2. In Anomalies, click Settings.
  3. In Alerts and notifications, click Create New Alert.
  4. In Anomaly Alerts, in Overview, from the Select Perspective drop-down list, select a Perspective on which you wish to create a notification alert.
  5. Click Save and continue.
  6. In Anomaly alert channels, click Add Channel.
  7. In the Select channel drop-down list, select Slack Webhook URL.
  8. Enter the Slack Webhook URL of the channel to which you want to send the notification alert.
    1. Follow the steps in Slack documentation for creating a Slack app, selecting your channel, and creating a webhook:  Sending messages using Incoming Webhooks.
  9. Once you're done, click Save and Continue. Your notification method is added.

Set Up Email Notifications

Perform the following steps to add your email to receive a notification alert.

  1. In Cloud Costs, click Anomalies.
  2. In Anomalies, click Settings.
  3. In Alerts and notifications, click Create New Alert.
  4. In Anomaly Alerts, in Overview, from the Select Perspective drop-down list, select a Perspective on which you wish to create a notification alert.
  5. Click Save and continue.
  6. In Anomaly alert channels, click Add Channel.
  7. In the Select channel drop-down list, select Email.
