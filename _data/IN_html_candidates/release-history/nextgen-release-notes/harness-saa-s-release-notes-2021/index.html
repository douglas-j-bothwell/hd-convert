Minor Release 0.280.10

Release Date: December 22, 2021

We're pleased to present Harness SaaS Release 0.280.10.

New Features and Enhancements

The following new features are added to the Harness SaaS components:

  • You can now perform the following on Perspectives:
    • Filter Perspectives based on Cloud Providers
    • Filter default OOTB Perspectives
    • Access Recently Edited Perspectives (CCM-5260)
    • See Create Cost Perspectives
  • In Harness CCM Budgets you can now:
    • Define budget over multiple periods
    • Define the start period for a budget
    • Add multiple budgets for a single Perspective
    • View confirmation pop-up when a budget is deleted (CCM-5232)
    • See Create a Budget

Issues Fixed

  • Build Bitbucket with tag was not working. (CI-3139, ZD-20670,ZD-21315)
  • Retry action on Initialize step to build CIE K8s pod was not working with Stage Failure Strategy.(CI-2954, ZD-20678)

Minor Release 0.279.9

Release Date: December 16, 2021

We're pleased to present Harness SaaS Release 0.279.9.

New Features and Enhancements

The following new features are added to the Harness SaaS components:

  • The health status checks for the Cloud Cost Management (CCM) AWS, Azure, and GCP connectors is automated. An hourly periodic check is introduced to monitor the connectivity status of the connectors. The error messages are also enhanced. (CCM-5178)
  • If your Azure billing export contains the Azure usage and charges information, then the following details are listed in the Perspectives:
    • Subscription name: Name of the subscription.
    • Publisher name: Publisher for Marketplace services.
    • Publisher type: Type of publisher. Break down Microsoft/Azure, Marketplace, and AWS costs.
    • Reservation ID: Unique identifier for the purchased reservation instance. Azure Reservations help you save money by committing to one-year or three-year plans for multiple products. Committing allows you to get a discount on the resources you use. See Manage Reservations for Azure resources.
    • Reservation name: Name of the purchased reservation instance.
    • Frequency: Indicates whether a charge is expected to repeat. Charges can either happen once (OneTime), repeat on a monthly or yearly basis (Recurring), or be based on usage (UsageBased).
    • See Analyze Cost for Azure Using Perspectives (CCM-5164, ZD-19546)

Issues Fixed

  • Nothing new in this release.
