The chaosctl command-line tool helps you manage Harness Chaos Engineering. You can use chaosctl to create Harness Chaos Engineering Delegates, Projects, and manage multiple accounts.

Usage

For detailed information on using chaosctl, including a complete list of chaosctl operations, see the chaosctl reference documentation.

  • Non-Interactive mode
  • Interactive mode

Requirements

The chaosctl CLI requires the following:

  • kubeconfig: chaosctl needs the kubeconfig of the Kubernetes cluster hosting Harness Chaos Engineering Delegates. The CLI currently uses the default path of kubeconfig, ~/.kube/config.
  • kubectl: chaosctl is using kubectl under the hood to apply manifests. To install kubectl, go to kubectl.

Installation Steps

Follow the steps for your operating system.

Linux/MacOS
  1. Extract the binary:
tar -zxvf chaosctl-<OS>-<ARCH>-<VERSION>.tar.gz
  1. Provide necessary permissions:
chmod +x chaosctl
  1. Move the chaosctl binary to /usr/local/bin/chaosctl.
Make sure to use root user or use sudo as a prefix.
mv chaosctl /usr/local/bin/chaosctl
  1. Run the chaosctl command in Linux/macOS:
chaosctl <command> <subcommand> <subcommand> [options and parameters]
Windows
  1. Extract the binary from the zip file using any extraction tool.
  2. Run the chaosctl command in Windows:
chaosctl.exe <command> <subcommand> <subcommand> [options and parameters]
  1. Check the version of the chaosctl:
chaosctl version
