In a Docker Connected On-Prem Setup, you can use the scripts described here to stop and restart Harness microservices. For example, if one or more boxes (or their Docker daemons) restart, you can use the Start Harness script to restart Harness microservices on one or all boxes. Here are the steps:

  1. SSH to the box where Harness Ambassador is running.
  2. Switch to the user that is running Harness Ambassador.
  3. Change directory to the harness-ambassador-scriptsfolder:
    cd ~/harness-ambassador-scripts
  4. Execute the script you need.

    To stop Harness microservices, run:
    bash stop_harness.sh

    To restart Harness microservices, run: 
    bash start_harness.sh
