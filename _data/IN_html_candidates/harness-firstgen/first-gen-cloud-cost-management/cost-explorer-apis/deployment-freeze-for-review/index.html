Step 1: Create a Project

As a first step, you need to create a project in Harness.

  1. In Harness, click Feature Flags and click Create Project.
  2. In Projects, click Project.
  3. In About the Project, in Name, enter the name for your project.
  4. Select color for your project.
  5. Select the organization from the list.
  6. Click Save and Continue.
  7. (Optional) In Invite Collaborators, in Invite People to Collaborate, add members to collaborate.
  8. (Optional) Assign a role to the collaborators and click Add. You can select Project Admin, Project Member, or Project Viewer.
  9. Once you're done adding contributors and adding a role, click Save and Continue.
  10. In Which Harness modules would you like to enable for this project?, enable CONTINUOUS Features.

Step 2: Create an Environment

Once you've created a project, you need to create an environment. An environment is the representation of your production or non-production infrastructure.

  1. In Feature Flags, select the Project that you created. For example, FFDOcs.
  2. In your Project, in Environments, click Environment.
  3. Enter a name for your environment.
  4. Select the environment type and click Create.

Step 3: Create an SDK Key

Now you have the environment ready, you need to create an SDK key for your environment.

  1. Click on the environment that you created in the previous step.
  2. Click Add Key.
  3. In Create SDK Key, in Name, enter a name for your SDK key.
  4. In Key Type, select Server and click Create. Select Client if you need to expose your feature flag to the client SDK.

    If your organization needs to rotate the keys, you can choose to add more than one key for the same environment.The SDK keys with the secret are created.
  5. Copy and store your secret.
    Secrets are visible right after the creation and redacted once you leave the page. Make sure to copy and store your secret.

    You need the secret when integrating the feature flag with your SDK.

Step 4: Create a Flag Type

Continuous Features support Boolean and Multivariate flag type.

Create a Boolean Flag Type

A Boolean flag typically has two variations true or false. Perform the following steps to create a boolean flag type:

  1. In Feature Flags, click Flag.
  2. Select the Boolean type.
  3. In About the Flag, in Name, enter a name for your boolean flag and click Next.
    If you need to make the feature flag a permanent flag, select the checkbox This is a permanent flag.
  4. In Variation settings, do the following:
    1. Select the flag type: In Flag Type, select Boolean.
    2. Define the variation: In boolean, you can have only two variations true or false.
      1. Enter a name for your variations. For example, True and False.
    3. Define rules for the flag: Defines which variation you will see by default when the flag is ON or OFF.
      Once you save the variations, a flag is created for the environment with these variations set as ON and OFF.

      For example,

      If the flag is ON, serve the variation True.
      If the flag is OFF, serve the variation False.
  5. Click Save and Close. The feature flag is created. By default, the feature flag is set to OFF.

Once you have created your boolean flag type, optionally you can modify the following settings of your flag:

  • (Optional) Edit Variations of the Feature Flag
  • (Optional) Add Prerequisites to the Feature Flag
  • (Optional) Edit Rules of the Feature Flag

Create a Multivariate Flag Type

Multivariate means that you can serve multiple variations of a feature to different user segments. Multivariate flags have multiple settings that allow you to define different variations using, strings, numbers, and JSON. Perform the following steps to create a multivariate flag type:

  1. In Feature Flags, click Flag.
  2. Select the Multivariate type.
  3. In About the Flag, in Name, enter a name for your multivariate flag and click Next.
    If you need to make the feature flag a permanent flag, select the checkbox This is a permanent flag.
  4. In Variation settings, do the following:
    1. Select the flag type: In Flag Type, select Multivariate and then select the type for your multivariate flag. You can select, STRING, JSON, or NUMBER.
    2. Define the variation: Enter a value and name for your variation. For example, "name":"CF" and name.
    3. Define rules for the flag: Defines which variation you will see by default when the flag is ON or OFF.

      Once you save the variations, a flag is created for the environment with these variations set as ON and OFF.

      For example,

      If the flag is ON, serve the variation name.
      If the flag is OFF, serve the variation email.

Once you have created your boolean flag type, optionally you can modify the variations, rules, and add prerequisites of your flag.

Step 5: Add Target to Your Feature Flag

You can add a target using Harness UI. Alternatively, you can add a target and define its attribute in your SDK code directly.

The targets added in your SDK are discovered automatically and populated in the Harness UI.

Targets are used to control which user sees the variation of a feature flag. A target can be any attribute that can be uniquely identified with an entity. For example, an application, systems, services, machines, resources uniquely identified by an IP address, email ID, user ID, etc.

Targets can be used to add users (individual or segment) for internal testing or beta program before a broader rollout.

Add a Target

You can add individual targets or upload a list of targets for a particular flag variation.

Add Individual Target(s)

The Individual option allows you to assign individual users to a particular flag variation.

  1. In Feature Flags, in Targets, click Target(s).
  2. Select Add a target.
  3. In Name, enter a name for your target.
  4. In Identifier, enter a unique identifier for your target.

    You can add multiple targets. Click + to add more targets.
  5. Click Add.
Upload a List of Targets

This option allows you to export a list of users in CSV format.

  1. In Feature Flags, click Targets.
  2. In Individual, click Target(s).
  3. Select Upload a list of targets.
  4. Upload your CSV file and click Add.

Create a Segment

  1. In Feature Flags, click Targets.
  2. In Segments, click Segment.
  3. In Create a Segment, enter a name for your segment and click Create.

(Optional) Edit Variations of the Feature Flag

You can edit the variations of your feature flag later as well.

  1. Click on the feature flag for which you want to edit the variation.
  2. Click the edit button.
  3. Edit the variations and modify the default rules. Once you're done, click Save.

(Optional) Add Prerequisites to the Feature Flag

You can add prerequisites for the initialization of the feature flag. The feature flag won't be initialized until the prerequisites are met.

  1. Click on the feature flag for which you want to add the prerequisites.
  2. Expand Prerequisites(what’s required before enabling flags) and click Prerequisites.
  3. In Add Prerequisites, click Prerequisites.
  4. Select a flag and variation from the drop-down list and click Save.

The added prerequisites are listed.

(Optional) Edit Rules of the Feature Flag

You can edit the default rules and also add custom rules to your feature flag. Custom rules serve specific targets differently.

  1. Click on the feature flag for which you want to edit the rules.
  2. In Targeting, click Edit Rules. You can edit the default and custom rules both.

    Default Rules

    Using default rules, you can specify to serve a specific variation, or apply a percentage rollout of a variation.
  3. In Default Rules, modify the rules. You can change the variation for your flag.
  4. If you want to do a percentage rollout, select % Percentage Rollout from the drop-down list. A percentage rollout allows choosing a percentage of users to see each variation of the flag. You can increase or decrease the percentage over time.
  5. In Bucket by, select Identifier or Name.
  6. Allocate percentages for different variations.
  7. Once you're done, click Save.

Custom Rules

Custom rules get enabled only when you click the Edit Rules button. Custom rules serve specific targets differently. In Custom Rules, you can add the following rules:

  • Serve Variation to Target
  • On Request, check for condition and serve variation
Serve Variation to Target
  1. Click Serve Variation to Target to select the variation for the specific target.
  2. From the Serve drop-down list, select the variation.
  3. Click + to specify the target to serve the variation.
  4. Select the target and click Done.The Custom Rule is added. You can add multiple Serve Variation to Target custom rule to serve different variations for different targets.
  5. Click Edit to modify the target to serve the variation.
  6. Once you're done, click Save.
On Request, check for condition and serve variation

This option allows you to add a condition to the variation and define custom rules based on the attributes of the Targets.

  1. Click On Request, check for condition and serve variation to add a condition to the variation for the specific target.
  2. From the On Request drop-down list, select the Identifier or Name.
  3. Enter attribute for the target.
  4. From the serve drop-down list, select the variation. You can choose to specify multiple conditions to serve a variation.
  5. Once you're done, click Save.

Integrate the Feature Flag with Java SDK

After creating your feature flag, you need to integrate the feature flag with the Java SDK code. To do that, as a first step you need to add Java dependency for the Java SDK.

Step 1: Add Maven Dependency in the pom.xml File

Add the following dependency in the pom.xml file:

<dependency>
  <groupId>io.harness.cf</groupId>
  <artifactId>cf-client-sdk-java</artifactId>
  <version>1.0.0-SNAPSHOT</version>
</dependency>

Step 2: Initialize the Continuous Feature Client

You need to have a client to communicate with the feature flag server. To do so, you need to initialize the Continuous Feature (CF) client.

Use the following example to initialize the CF client. The base URL is the endpoint of the server where you created your environment.

CfClient cfClient = new CfClient(apiKey);

Step 3: Enter the SDK API Keys

You need to enter the SDK API keys that you created for your environment. The SDK keys authorize your application to connect to the CF client. See Step 3: Create an SDK Key.

Step 4: Define Target

You can add a target and define its attribute in your code. The targets added in your SDK are discovered automatically and populated in the Harness UI.

Sample Code

Target target = Target.builder().name("User1").identifier("Harness").build();

Sample Code Using Custom Attributes

Target target =
      Target.builder()
          .name("target1")
          .identifier("target1-identifier")
          .attributes(
              new ImmutableMap.Builder<String, Object>()
                  .put("accountId", "kmpySmUISimoRrJL6NL73w")
                  .put("name", "ObjectName5")
                  .put("licenseType", "TRIAL")
                  .build())
          .build();

Step 4: Evaluate Target for Your Feature Flag

A feature flag is evaluated for a particular target.

boolean result = cfClient.boolVariation(<feature_identifier>, target, <default_result>);

For example:

boolean result = cfClient.boolVariation("first_flag", target, false);

Sample Code for Java SDK

Here is a sample code for integrating your feature flag with the Java SDK.

public class HelloWorld {
  public static void main(String[] args) throws Exception {
    String apiKey = "e26d2a3c-8f3e-4c44-1234-96051ccaa1a4";
    CfClient cfClient = new CfClient(apiKey);

    Target target = Target.builder().name("User1").identifier("user1@example.com").build();

    while (true) {
      Thread.sleep(2000);
      boolean result = cfClient.boolVariation("toggle", target, false);
      System.out.println("Boolean variation is "+ result);
    }
  }



}

Integrate the Feature Flag with Golang SDK

Once you have your feature flag ready, integrate the flag with the Golang SDK code.

Step 1: Install the Golang Feature Flag SDK

go get github.com/wings-software/ff-client-sdk-go

Step 2: Import the Library

import harness "github.com/wings-software/ff-client-sdk-go/pkg/api"

Step 3: Create Client Instance

Create client instance for setting the communication with the SDK.

client := harness.NewClient(sdkKey)

Step 4: Add Target to Your Feature Flag

Define the target for your feature flag. A feature flag is evaluated for a particular target.

target := dto.NewTargetBuilder("key").
 		Firstname("John").
 		Lastname("doe").
 		Email("johndoe@acme.com").
 		Country("USA").
 		Attribute("height", 160).
 		Build()

Step 5: Evaluate Feature Flag

showFeature, err := client.BoolVariation(featureFlagKey, target, false)

Sample Code for Golang

Here is a sample code for integrating your feature flag with the Golang SDK:

package main

import (
	"context"
	"fmt"
	"log"
	"time"

	harness "github.com/wings-software/ff-client-sdk-go/pkg/client"
	"github.com/wings-software/ff-client-sdk-go/pkg/dto"
)

const sdkKey = "7bc7df92-16bc-4c62-9ad4-b78a497690e7"

const featureFlagKey = "toggle"

func main() {

	client, err := harness.NewCfClient(sdkKey,
		harness.WithUrl("http://34.82.119.242/api/1.0/"),
		harness.WithEventSourceUrl("http://34.82.119.242/api/1.0/stream/environments/%s"),
		harness.WithStreamEnabled(true),
	)
	defer func() {
		if err := client.Close(); err != nil {
			log.Printf("error while closing client err: %v", err)
		}
	}()

	if err != nil {
		log.Printf("could not connect to CF servers %v", err)
	}

	target := dto.NewTargetBuilder("john").
		Firstname("John").
		Lastname("Doe").
		Email("john@doe.com").
		Country("USA").
		Custom("height", 186).
		Build()

	ctx, cancel := context.WithCancel(context.Background())

	go func() {
		for {
			select {
			case <-ctx.Done():
				return
			default:
				showFeature, err := client.BoolVariation(featureFlagKey, target, false)

				if err != nil {
					fmt.Printf("Error getting value: %v", err)
				}

				fmt.Printf("KeyFeature flag '%s' is %t for this user\n", featureFlagKey, showFeature)
				time.Sleep(10 * time.Second)
			}
		}
	}()

	time.Sleep(5 * time.Minute)
	cancel()
}
