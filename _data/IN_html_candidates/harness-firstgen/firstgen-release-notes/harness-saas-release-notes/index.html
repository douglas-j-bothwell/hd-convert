Minor Release 72400

Release Date: December 22, 2021

We're pleased to present Harness SaaS Release 72400.

General What's New and Early Access

  • New features added to Harness.
  • Features behind Feature Flags (Early Access).

New Features and Enhancements

The following new features are added to the Harness SaaS components:

  • You can now use a specific AWS role like AWS Cloud Provider, Region, or Role ARN when provisioning a Terraform Apply step to perform Terraform operations in your Workflow. These options allow you to use different roles for different Terraform steps. This feature is currently behind the Feature Flag TERRAFORM_AWS_CP_AUTHENTICATION. Contact Harness support to enable the feature.(CDP-17256, ZD-14432, ZD-20339)
    • AWS Cloud Provider: The AWS Cloud Provider is used for authentication.
    • Region: The AWS region where you will provision your resources. If no region is specified, Harness uses us-east-1.
    • Role ARN: Enter the Amazon Resource Name (ARN) of an AWS IAM role that Terraform assumes when provisioning. This allows you to tune the step for provisioning a specific AWS resource.
    • See Provision using Terraform Provision Step, Using Terraform Apply Command, and Remove Provisioned Infra
  • ​You can now start a deployment Pipeline after CI build is completed with charts that are not collected by Harness, and without having to wait for Harness to pull the Helm chart version. When you fire a webhook trigger or a GraphQL API to start an execution, Harness will manually pull the Helm Chart. The deployment will proceed only if the Helm chart is found and collected. Harness stores the collected chart from that point. (CDC-15686, ZD-19121)
  • You can now perform the following actions on Workflows which are part of Pipelines if you have execute pipeline permission:
    • Abort
    • Pause
    • Manual intervention

Workflows which are not part of any Pipelines continue to be governed by execute workflow permission. (CDC-14155, ZD-7478, ZD-16369)

Issues Fixed

  • HTTP Step was failing with error `Invalid Request: Delegate is configured not to use proxy for the given url`. This happened when there was a proxy set on Delegate and the URL provided in HTTP Step was added as no_proxy.(CDC-16289, ZD-20959)
  • Pipelines with multiple stages in parallel were failing during Pipeline execution, with workflow being grayed out. (CDC-16042, ZD-20466)
  • Using Delegate selectors with custom Shell Script approval resulted in corrupt workflows and pipelines. (CDC-15205, ZD-20011, ZD-20527, ZD-21104)

Minor Release 72300

Release Date: December 16, 2021

We're pleased to present Harness SaaS Release 72300.

General What's New and Early Access

  • New features added to Harness.
  • Features behind Feature Flags (Early Access).

New Features and Enhancements

The following new features are added to the Harness SaaS components:

  • You can trigger Harness Workflow and Pipeline deployments on a time schedule. You can select how often to execute the Trigger by hour, days, etc.

  • If your Azure billing export contains the Azure usage and charges information, then the following details are listed in the Perspectives:
    • Subscription name: Name of the subscription.
    • Publisher name: Publisher for Marketplace services.
    • Publisher type: Type of publisher. Break down Microsoft/Azure, Marketplace, and AWS costs.
    • Reservation ID: Unique identifier for the purchased reservation instance. Azure Reservations help you save money by committing to one-year or three-year plans for multiple products. Committing allows you to get a discount on the resources you use. See Manage Reservations for Azure resources.
    • Reservation name: Name of the purchased reservation instance.
    • Frequency: Indicates whether a charge is expected to repeat. Charges can either happen once (OneTime), repeat on a monthly or yearly basis (Recurring), or be based on usage (UsageBased).
    • See Analyze Cost for Azure. (CCM-5164, ZD-19546)
