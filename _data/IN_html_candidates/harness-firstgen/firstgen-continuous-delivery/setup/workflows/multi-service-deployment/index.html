Multi-Service deployment allows you to deploy multiple services in the same workflow, to one or more infrastructures. Similar to a Canary deploy, this type of deployment comprised of multiple phases. Each phase is essentially similar to a single basic deployment, where Harness deploys a service (or services) to a defined number of nodes. Combined together, several phases allow you to deploy your service(s) in a controlled, gradual, and verified process. Since each phase is independent, you can add verification and custom commands before moving on to the next phase.

 

  1. Pre-deployment Steps: You can add pre-deployment steps prior to deploying your code. This is similar to the Add Command option.
  2. Deployment phases: Click on “Add Phase” in order to configure your first batch. Then choose which service you’d like to deploy during this phase, and to which infrastructure. You can also add variables under the “Service Variable Overrides” section, and these variables will override any other variables that were previously configured for this service. Next, in order to configure the phase itself, follow the instructions for configuring Basic Deployment.
  3. Post-deployment Steps: Similar to the Pre-deployment steps, you can add additional commands after your code was deployed.
