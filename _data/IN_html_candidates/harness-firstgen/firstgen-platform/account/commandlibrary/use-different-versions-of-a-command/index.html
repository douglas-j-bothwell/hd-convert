Each time you edit a command you create a new version of it. You can use different versions of a command in your Services and Workflows.

Some commands in the Command Library have multiple versions. If you have downloaded multiple versions of a command, you can select which version to use as the default version. You can make a specific version of the command as the default version.

When a user links to the command from a Service or Workflow, the default version is used.

Step: Update the Default Version

To update the default version of a command, do the following:

  1. In Setup, click Template Library and then click Downloads. The downloaded commands are listed.
  2. Select the command for which you want to update the default version.
  3. Click on the Version drop-down and select the version that you want to make as the default version.
  4. Click Select as default for the selected version. The default version is updated.

Step: Use Different Versions

To use versions of a command, do the following:

  1. Open a Service or Workflow containing a linked command. A link icon is next to the command entity.
  2. To select a different version of the command, click the vertical ellipsis, and click Manage Versions.The Manage Version settings appear.
  3. In Default Version, click the version, locate the version you want to use, and then click select. Click Submit. The new version is displayed next to the linked command name.
